# 01 — REQUIRED ACCESS
**Status:** IMMUTABLE v1 · READ-ONLY  
**Audience:** All VAs  

---

## Access Philosophy

You are given the **minimum access required** to do your job safely.

If you do not have access:
- do not work around it
- do not borrow credentials
- do not guess

---

## Required Systems

### GitHub (Read-Only)
You must be able to:
- view the repository in a browser
- browse files

You may NOT:
- push code
- upload files
- edit documentation

---

### Cloudflare (Role-Based)
If required for your role, you will be given:
- limited permissions
- scoped access

**Important:** Environment variables (secrets) are configured in Cloudflare Pages by the Owner or a Technical VA only.

---

### Airtable (Owner / Technical VA only)
This repo can store **Request Assistance** submissions in Airtable.

Most VAs do **not** need Airtable access.

If your role requires it, you will be granted:
- access to a single Airtable base
- no ability to share public links

Never:
- paste Airtable tokens into chat
- store tokens in repo files

---

## How to Request Access

Email the Owner:

**To:** info@spryvc.com  
**Subject:** Access Request — Listings System  

Include:
- your role (Standard VA / Technical VA)
- system requested (GitHub / Cloudflare / Airtable)
- reason access is needed

---

## If Access Is Missing

Do not proceed.
Escalate immediately.
